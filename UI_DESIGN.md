# Audiobookshelf iOS UI 设计规范

本设计规范旨在为 Audiobookshelf iOS 客户端提供一套清晰、简洁且现代的视觉设计方案。设计遵循 Apple Human Interface Guidelines，并针对有声书/播客场景进行了优化。

## 1. 设计系统 (Design System)

### 1.1 色彩体系 (Color Palette)

采用现代、清爽的色彩风格。**主色调**选用“电光紫 (Electric Indigo)”，既体现科技感又保持阅读/听书的沉浸感。

| 颜色名称 | 变量名 | 亮色模式 (Light) | 深色模式 (Dark) | 用途 |
| :--- | :--- | :--- | :--- | :--- |
| **Primary** | `Color.accentColor` | `#5856D6` (Indigo) | `#5E5CE6` (Indigo Light) | 按钮、链接、高亮图标、进度条 |
| **Background** | `Color.systemBackground` | `#FFFFFF` | `#000000` | 页面主背景 |
| **Secondary BG** | `Color.secondarySystemBackground` | `#F2F2F7` | `#1C1C1E` | 卡片背景、列表分组背景 |
| **Tertiary BG** | `Color.tertiarySystemBackground` | `#FFFFFF` | `#2C2C2E` | 模态视图、浮层背景 |
| **Text Primary** | `Color.label` | `#000000` | `#FFFFFF` | 主要标题、正文 |
| **Text Secondary**| `Color.secondaryLabel` | `#3C3C43` (60%) | `#EBEBF5` (60%) | 次要信息、作者名、说明文字 |
| **Separator** | `Color.separator` | `#3C3C43` (29%) | `#545458` (65%) | 分割线、边框 |
| **Error** | `Color.red` | `#FF3B30` | `#FF453A` | 错误提示、删除操作 |
| **Success** | `Color.green` | `#34C759` | `#30D158` | 下载完成、操作成功 |

### 1.2 字体排印 (Typography)

使用 iOS 系统字体 **SF Pro**。

| 样式名称 | 字重 (Weight) | 大小 (Size) | 用途 |
| :--- | :--- | :--- | :--- |
| **Large Title** | Bold | 34pt | 页面主标题 (如“首页”) |
| **Title 2** | Bold | 22pt | 分区标题 (如“继续收听”) |
| **Headline** | Semibold | 17pt | 卡片标题、书名 |
| **Body** | Regular | 17pt | 正文内容 |
| **Subheadline** | Regular | 15pt | 列表副标题 |
| **Footnote** | Regular | 13pt | 辅助信息、元数据 |
| **Caption 1** | Regular | 12pt | 标签栏文字、微小说明 |

### 1.3 间距与圆角 (Spacing & Radius)

*   **全局栅格**: 4pt 为基准，常用间距: 8pt, 16pt, 24pt, 32pt。
*   **圆角 (Corner Radius)**:
    *   卡片/封面: 12pt
    *   按钮: 10pt 或 Capsule (全圆角)
    *   模态弹窗: 16pt

### 1.4 主题模式 (Theme)

支持三种显示模式：
* 跟随系统
* 浅色
* 深色

主题选择入口位于“个人中心/外观”，偏好会本地持久化，应用全局生效。

### 1.5 按钮体系 (Buttons)

统一的按钮类型与尺寸，确保跨界面一致性：
* **主按钮 (Primary)**: 用于最重要的操作（如“播放”/“登录”）
  * 形态：圆形或宽按纽；圆形直径 60pt；宽按纽高度 50pt，最小宽度 220pt
  * 颜色：背景使用 `Primary`，文字/图标为白色
* **次按钮 (Secondary)**: 辅助操作（如“下载”、“章节”）
  * 尺寸：44x44（图标按钮）；或高度 44pt（文字/图标组合）
  * 样式：边框按钮 `.bordered` 或浅色填充 `Secondary BG`
* **图标按钮 (Icon Only)**: 统一触控目标 44x44，图标大小 22pt
* **按钮间距**: 同一操作区内水平间距 14pt；与其他模块垂直间距 16pt
* **状态**: 默认 / 悬停（iPad 鼠标）/ 按下 / 禁用 / 加载；禁用态降低不透明度至 40%；加载态显示微型环形进度指示

### 1.6 图标与尺寸 (Icons)

* 统一使用 SF Symbols，大小随按钮类型匹配：
  * 主操作图标：22pt（置于 60pt 圆形内）
  * 次操作图标：20–22pt（置于 44x44 按钮内）
* 播放器常用图标：`play.fill`、`pause.fill`、`backward.fill`、`forward.fill`、`list.bullet`、`moon`、`airplayaudio`

## 2. 界面详细设计 (Screen UI Specifications)

### 2.1 登录流程 (Authentication)

简洁明了，减少认知负担。

*   **AuthView**:
    *   **布局**: 顶部为 App Logo 与 App 名称；下方为卡片化表单（服务器配置 / 服务器 / 账号）。
    *   **服务器配置卡片**:
        *   配置选择：使用菜单式 Picker 展示服务器配置名称。
        *   管理入口：右侧“管理”按钮进入服务器管理页（新增/编辑/删除）。
        *   地址预览：在卡片内展示根据当前输入拼装的 Base URL（可复制），便于快速核对。
    *   **协议选择**: HTTP / HTTPS 以分段控制 (Segmented) 并排单选，默认选中 HTTPS。
    *   **输入框**: 高度 50pt，圆角 10pt；卡片背景使用 `Secondary BG`，输入框背景使用 `Tertiary BG`。
    *   **登录按钮**: 主按钮使用 `Primary`，点击后自动执行“服务器联通性验证 + 登录认证”，无需单独的检查按钮。
    *   **错误反馈**: 登录失败或服务器不可达时，在登录按钮下方展示红色错误信息（Footnote）。

### 2.2 首页 (Home View)

以内容为核心，强调沉浸感。

*   **Header**: 显示问候语（如“早上好”），右侧为用户头像（点击进入设置）。
*   **Section: 继续收听 (Continue Listening)**:
    *   **样式**: 横向滚动列表 (ScrollView .horizontal)。
    *   **卡片**: 宽 280pt，高 120pt。左侧为封面 (80x80)，右侧为标题、作者、进度条。
    *   **进度条**: 位于卡片底部，高 4pt，已播放部分为 `Primary` 色。
    *   **背景**: `Secondary BG`，圆角 12pt，轻微阴影。
*   **Section: 最近添加 (Recently Added)**:
    *   **样式**: 横向滚动列表。
    *   **卡片**: 竖向排列。封面图 (140x140) + 下方标题 (截断 2 行) + 作者 (1 行)。
    *   **交互**: 点击封面跳转详情。

### 2.3 媒体库 (Library View)

高效的筛选与浏览。

*   **Top Bar**:
    *   左侧: 库切换器 (LibraryPicker) - 下拉菜单样式，文字加粗，旁边有向下箭头。
    *   右侧: 筛选 (Filter) 和 排序 (Sort) 图标按钮。
*   **Filter Bar** (展开时):
    *   水平滚动的胶囊型标签 (Capsule Chips)。选中状态背景为 `Primary`，未选中为 `Secondary BG`。
*   **Grid Content**:
    *   `LazyVGrid`，自适应列数 (Adaptive, min: 160pt)。
    *   **封面**: 保持 1:1 或原比例，圆角 8pt。
    *   **元数据**: 封面右下角显示时长或“已读”标记 (Badge)。

### 2.4 详情页 (Item Detail View)

信息的层级展示与操作便利性。

*   **Hero Section**:
    *   背景: 移除背景图与模糊效果，使用 `Background` 纯色，保持简洁与可读性。
    *   封面: 居中显示，投影 (Shadow) 半径 20pt，增加立体感。
    *   间距: 顶部留白 24pt；封面与标题间距 6pt。
*   **Info Section**:
    *   标题: Title 2，居中，多行显示。
    *   作者: Body，灰色，居中。
*   **Action Bar**:
    *   主操作: 播放/继续播放（圆形 60pt，`Primary` 背景，白色图标）
    *   次操作: 收藏（图标按钮 44x44，`Secondary BG` 背景，图标 `Primary`）
    *   章节入口: 文字+图标按钮，高度 44pt，`.bordered` 样式
*   **Content**:
    *   简介: 默认显示 3 行，点击“更多”展开；卡片背景使用 `Secondary BG`，圆角 12pt，左右内边距 16pt。
    *   章节列表: 简单的列表项，包含章节号、标题、时长。正在播放的章节高亮显示。

### 2.5 播放器 (Player UI)

全屏沉浸式体验。

*   **背景**: 动态模糊背景 (封面主色)。
*   **Layout**:
    *   顶部: 抓手条 (Grabber)，向下箭头收起。
    *   中部: 大封面 (带投影)。
    *   信息: 标题 (滚动跑马灯效果 if needed)，作者。
    *   **进度条**: 
        *   滑块 (Thumb) 略微放大，方便拖拽。
        *   两端显示“当前时间”和“剩余时间” (倒计时格式)。
    *   **控制区**:
        *   主控: 快退 15s | 播放/暂停 (大) | 快进 30s。
        *   辅控 (底部): 播放速度 (1.0x) | 睡眠定时 (月亮) | 章节 (列表) | AirPlay。
        *   **加载态**: 缓冲/加载媒体资源时，播放按钮显示转圈并提示“正在加载媒体资源…”，仍允许用户点击暂停与调整倍速。
    *   **播放偏好**:
        *   默认倍速会被记忆并在下次启动 App 时自动恢复。
        *   “跳过片头片尾”在“我的/播放”中配置：可开关、可分别设置每章片头/片尾跳过秒数。
        *   开启后：章节开始自动跳过片头；章节尾部剩余时间到达片尾配置时自动切到下一章。
*   **Mini Player**:
    *   高度 60pt。悬浮在 TabBar 上方 1pt（不遮挡 TabBar 点击区域）。
    *   背景: `Tertiary BG` (Material 材质)。
    *   布局: 封面 (40x40) | 标题(截断) + 章节名(可水平滚动) | 当前/总时长(等宽数字) | 播放/暂停按钮 (44x44) | 关闭按钮 (可选)
    *   进度条: 位于顶部边沿，高度 2pt，极简。
    *   交互: 点击任意空白区域展开为全屏播放器；左右滑动切换上一/下一章节（可选）
    *   章节信息: 副标题展示章节名（可滚动）；时间信息独立区域固定展示。

### 2.6 搜索 (Search View)

*   **搜索框**: 标准 UISearchBar 样式。
*   **默认状态**: 显示“热门搜索”或“最近搜索”标签云。
*   **结果列表**:
    *   列表布局 (List)。
    *   左侧小封面 (50x50)，右侧标题、作者、类型 (Book/Podcast)。
    *   右侧尾部操作：快速播放（44x44 图标按钮）、更多菜单（三点）

### 2.7 个人中心 (Profile View)

*   **Header**: 用户头像、用户名、服务器地址。
*   **List**:
    *   下载管理 (Downloads)
    *   收听统计 (Stats)
    *   设置 (Settings)
    *   注销 (Logout) - 红色文字。
*   **播放设置**:
    *   默认倍速：下拉选择 0.75x/1.0x/1.25x/1.5x/2.0x。
    *   跳过片头片尾：开关 + 片头/片尾秒数 Stepper（步进 1s，范围 0–180s）。

## 3. App 图标方案 (App Icon Concepts)

提供 5 款不同风格的图标设计方案，供用户选择。

### 方案 A: "声波书韵" (The Sonic Book) - *默认推荐*
*   **设计**: 一个打开的书本轮廓，书页中间转化为律动的声波线条。
*   **颜色**: 渐变背景 (深蓝到紫)，白色线条。
*   **寓意**: 书籍与声音的完美融合，经典且直观。

### 方案 B: "极简播放" (Minimal Play)
*   **设计**: 一个圆润的三角形（播放符号）与一个矩形（书脊）巧妙结合，形成类似字母 "A" 或书本的抽象形态。
*   **颜色**: 纯白色背景，中心图形使用 `Electric Indigo` 主色。
*   **风格**: 极简主义，扁平化，适合现代 iOS 风格。

### 方案 C: "知识涟漪" (Knowledge Ripple)
*   **设计**: 中心为一个圆点（声源），向外扩散出几圈同心圆弧（涟漪），每一圈由微小的文字或线条组成，暗示故事的传播。
*   **颜色**: 深色夜空背景，霓虹蓝/紫色线条。
*   **风格**: 科技感，神秘感，适合深色模式爱好者。

### 方案 D: "有声书架" (The Shelf)
*   **设计**: 抽象化的书架侧视图，其中一本书倾斜着，书脊上带有耳机图标。
*   **颜色**: 温暖的木质色调或柔和的 pastel 色彩。
*   **风格**: 拟物风格的现代演绎 (Neumorphism)，给人温馨、私人的图书馆感觉。

### 方案 E: "首字母 A" (Typographic A)
*   **设计**: 艺术化的字母 "A" (代表 Audio/Audiobookshelf)。"A" 的横杠被替换为声波图形，或者 "A" 的形状由耳机线盘绕而成。
*   **颜色**: 鲜艳的橙红色或蓝绿色渐变。
*   **风格**: 强烈的品牌符号感，高辨识度。

## 4. 异步与性能策略 (Async & Performance)

面向流畅交互的统一逻辑，参考 audiobookshelf-app-master 的分层与同步策略：

* **请求状态模型**: `idle / loading / success / error` 四态统一；加载态显示小型进度，成功与错误采用轻量 Toast（不阻塞）
* **搜索防抖**: 输入后 300ms 防抖再发起搜索；快速连续输入会取消前一任务
* **播放进度同步**: 10 秒节流向服务端发送一次；暂停/停止时立即同步；字段统一为 `currentTime / duration / timeListened`
* **任务取消**: 使用 Swift Concurrency 的 `Task` + 协作取消；UI 更新在 `@MainActor`；进入新页面时取消上页后台请求
* **下载队列**: `URLSession` 后台配置；最大并发 2–3；显示环形进度；断点续传与失败重试（指数退避至最多 3 次）
* **章节缓存**: 内存 + 磁盘双缓存；打开章节列表先渲染缓存，后台对比更新，避免长列表阻塞
* **Skeleton 与占位**: 首页与库网格在数据未到时显示骨架卡片；图片使用渐进式加载与淡入动画
* **错误恢复**: 网络错误提供“重试”与“离线播放”入口（如本地已缓存）
* **电源与性能**: 长时间播放时降低 UI 刷新频率（如进度 UI 每 250ms 刷新一次），减少电量消耗

## 5. 动效与过渡 (Motion)

* **页面过渡**: 使用系统导航动画；大封面采用匹配几何效果（Hero）
* **控件反馈**: 按钮按下缩放 96% 并投影减弱；播放/暂停切换采用交错淡入淡出
* **Mini Player 展开**: 自底部向上弹性过渡，进度条连续动画不被中断

---

## 6. 可访问性 (Accessibility)

* 颜色对比度满足 WCAG AA（文字与背景对比度 ≥ 4.5:1）
* 支持动态字体大小（Dynamic Type）；换行截断优先保证可读性
* VoiceOver：关键按钮提供清晰的可读标签与状态说明（如“正在播放”）
* 触控目标最小 44x44；复杂操作提供替代路径（如上下文菜单）
